# 테스트 및 검증 기록

## 자동화 테스트

```sh
QT_QPA_PLATFORM=offscreen .venv/bin/pytest -vv
```

테스트 중 생성되는 입력 fixture와 변환 결과는 pytest의 시스템 임시 디렉터리를 사용한다.
저장소에 포함하지 않을 개인 문서·공식 대용량 fixture는 `local-test-data/`, 로컬 테스트
보고서는 `test-results/`에 두며 두 경로 모두 `.gitignore`로 제외한다.

최신 결과: **80 passed**.

검증 범위:

- GUI 생성과 버튼 상태
- 상단의 별도 폴더 추가 버튼 제거 및 파일·폴더 드래그 앤 드롭 유지
- 건너뜀 상태를 라이트·다크 모드 모두 오류색(빨강)으로 표시
- 건너뜀 항목이 진행률과 강아지 위치를 증가시키지 않는지 한글·영문에서 검증
- 단일 출력 폴더 선택, 선택 전 안내 문구와 선택 후 실제 저장 경로 표시
- 해·달 벡터 아이콘만 있는 슬라이드 스위치의 테마 전환, 전체 팔레트 전환, 설정값 저장
- 독립된 전체 너비 행에서 네 다리를 교차하며 달리는 옆모습 강아지 진행 바
- 외부 이미지나 시스템 이모지 없이 직접 그린 뼈 목표 아이콘
- 하단 상태 문구와 본문 요약 문구의 왼쪽 기준선 정렬 및 상·하 여백
- 라이트·다크 팔레트의 입력·텍스트·버튼·비활성 색상과 전체 컨트롤 상태 스타일
- 출력 폴더 자동 열기의 기본 비활성화, 설정 저장, 완료 시 동작
- 한국어판 회귀와 별도 영문판의 UI·상태·오류·설정 저장소 분리
- 영문 GUI → Scanner → 영어 Worker → Markdown 저장 전체 흐름
- GUI → Scanner QThread → JobManager → QProcess Worker → Markdown 저장 전체 흐름
- TXT, HTML, CSV, JSON, XML, RTF, PDF, DOCX, PPTX, XLSX, XLS, EPUB, ZIP 실제 변환
- 별도 프로세스 실패 후 나머지 작업 계속
- Worker 타임아웃과 강제 종료
- 실행 중/대기 중 작업 취소
- 실패 작업 재시도
- 지정 출력 폴더 미선택 차단
- NFC/NFD 한글 경로 중복 제거
- 숨김파일, AppleDouble, macOS 패키지, 심볼릭 링크 폴더 제외
- 0바이트, 미지원 형식, 누락 파일, 파일 크기/개수 상한
- 동일 이름 및 대소문자 충돌 자동 이름 변경
- 지정 폴더 상대 디렉터리 구조 유지
- 원자적 저장과 저장 실패 시 임시파일 정리
- 손상 PDF 및 DOCX 파일 시그니처 거부와 불완전 결과 미생성

## Microsoft 공식 fixture 교차 검증

Microsoft MarkItDown 공식 저장소의 테스트 파일을 이용해 개발 Worker와 완성된 앱 번들
Worker를 각각 검증했다.

- Outlook MSG
- DOCX
- PPTX
- XLSX
- XLS
- EPUB
- PDF

각 변환 결과에 공식 test vector의 필수 문자열이 포함되는지 확인했다.

## 앱 번들 검증

- PyInstaller arm64 `.app` 생성 성공
- 한국어 `Batch Markdown Converter Korean.app`과 영문 `Batch Markdown Converter English.app` 별도 생성 성공
- 번들 내부 Worker로 공식 fixture 7종 변환 성공
- 번들 내부 Worker가 손상 PDF/DOCX를 오류로 거부
- 실패 시 부분 `.md` 파일이 생성되지 않음
- `codesign --verify --deep --strict` 통과
- `Info.plist` 검사 통과
- Mach-O arm64 실행 파일 확인
- 사용하지 않는 Qt 모듈 제거 후 최종 앱 크기 약 228MB

테마 변경 후 Scanner 종료 수명주기를 보강했으며, 전체 35개 테스트를 3회 연속,
GUI E2E 테스트를 별도로 10회 연속 실행해 모두 통과했다.

출력 폴더 자동 열기 옵션 추가 후 전체 40개 테스트와 관련 설정·GUI·E2E 테스트
7개를 다시 실행해 모두 통과했다.

별도 영문판 추가 후 전체 44개 테스트와 영문 전용 테스트 4개를 통과했다. 완성된
영문 앱 번들 Worker의 정상 변환과 손상 PDF 영어 오류도 별도로 확인했다.

라이트·다크 테마를 전체 컨트롤 상태별 색상 체계로 교체한 후 전체 53개 테스트와
영문·테마·스위치 전용 테스트 13개를 통과했다. 두 테마의 메인 화면과 설정 창도
각각 렌더링해 입력창, 화살표, 스위치, 비활성 버튼, 테이블, 진행 바 대비를 확인했다.

진행 바를 오른쪽을 바라보는 옆모습 강아지가 퍼센트 위치까지 달리는 애니메이션으로
교체한 후 전체 57개 테스트를 통과했다. 0·35·70·100% 상태를 라이트·다크 모드로
각각 렌더링해 강아지 위치, 퍼센트 표시, 진행 색상, 뼈다귀 가시성을 확인했다.

상단 테마 전환을 160ms 손잡이 애니메이션이 있는 슬라이드 스위치로 교체한 후 전체
58개 테스트를 통과했다. 한글·영문 스위치를 라이트·다크 모드로 각각 렌더링해 트랙,
손잡이 위치, 레이블 및 인접 버튼의 대비를 확인했다.

강아지 진행 바와 뼈다귀를 합계·작업 버튼과 분리된 전체 너비 행으로 옮긴 후 전체
58개 테스트를 다시 통과했다. 한글 메인 화면을 라이트·다크 모드로 렌더링해 진행 바가
본문 너비의 80% 이상을 사용하고 합계 행과 겹치지 않는지 확인했다.

강아지를 정적 이모지에서 네 다리와 몸통을 직접 그리는 벡터 애니메이션으로 교체하고,
테마 스위치의 글자를 제거해 해·달 벡터 아이콘을 내부에 배치한 후 전체 61개 테스트를
통과했다. 네 보행 프레임과 두 테마의 스위치·전체 화면을 각각 렌더링해 확인했다.

출력 모드 선택을 제거하고 폴더 선택 버튼 하나로 단순화했다. 선택 전에는
안내 문구를, 선택 후에는 실제 저장 경로를 표시하고 모든 결과를 그 폴더에 저장한다.
하단 상태 문구에도 상·하 여백을 추가해 창 경계와 글자가 붙지 않도록 했다.

진행 바 강아지를 큰 머리, 짧은 다리, 크림색 얼굴·배, 볼터치, 말린 꼬리를 가진
통통한 시바·코기풍 벡터로 다시 디자인했다. 전체 64개 테스트를 통과했으며 네 보행
프레임과 라이트·다크 전체 화면을 렌더링해 크기, 표정, 다리 움직임을 확인했다.

## 언어별 배포 패키지 검증

버전 `0.1.0`의 한국어판과 영문판을 각각 독립된 arm64 DMG로 생성하고 다음 항목을
검증했다.

- `release/Korean/Batch-Markdown-Converter-Korean-0.1.0-arm64.dmg` 생성
- `release/English/Batch-Markdown-Converter-English-0.1.0-arm64.dmg` 생성
- 두 DMG의 SHA-256 체크섬 일치
- `hdiutil verify` 무결성 검사 통과
- 읽기 전용 마운트 및 내부 앱, `Applications` 바로가기, 언어별 안내문 확인
- 프로젝트 MIT 라이선스, 제3자 고지 및 번들 의존성 라이선스 포함 확인
- 한국어판과 영문판의 버전 `0.1.0` 및 서로 다른 번들 ID 확인
- 두 앱 모두 `codesign --verify --deep --strict` 통과
- 두 실행 파일 모두 Mach-O arm64 확인

현재 산출물은 ad-hoc 서명 상태이므로 로컬 사용과 제한된 테스트 배포용이다. 일반
사용자에게 경고 없는 공개 배포를 하려면 Developer ID 서명과 Apple 공증을 추가해야 한다.

프로젝트 이름을 `Batch Markdown Converter`로 확정한 후 창 제목, 앱 번들, 실행 파일, 번들 ID,
DMG, 체크섬, 설치 안내와 GitHub README의 이름 일관성을 자동 검증하도록 보강했다. 영문 앱
번들 Worker로 지원 형식 13종을 모두 실제 변환했고 손상 PDF가 실패하며 부분 결과를 남기지
않는 것도 확인했다.

간헐적으로 발생하던 PySide6 `QProcess` 네이티브 종료 충돌을 재현해, 완료 프로세스 정리 후
다음 이벤트 루프에서 작업을 이어가도록 수명주기를 수정했다. 재시도·취소·타임아웃·시작 실패
경로를 반복 스트레스 테스트하고 전체 테스트를 연속 실행해 통과했다.

개발 의존성의 `pytest`를 9.1.1로 갱신했으며 `pip check`와 `pip-audit` 결과 런타임 및 개발
환경에서 알려진 의존성 취약점이 발견되지 않았다. 로컬 프로젝트 자체는 공개 취약점 데이터베이스에
등록된 PyPI 패키지가 아니므로 감사 대상에서 제외되는 것이 정상이다.

공개 전 개인정보 감사에서 로컬 사용자명이 들어간 저작권·번들 ID를 발견해 프로젝트 기여자
명의와 중립 번들 ID로 교체했다. 제3자 라이선스 수집기가 `packaging/licenses/` 아래 Python
소스와 바이트코드까지 포함하던 문제도 수정해 법적 텍스트 파일만 복사하도록 제한했다. 한글·영문
앱 번들과 최종 DMG를 바이트 단위로 재검사해 로컬 사용자명과 홈 경로가 없고, 라이선스 묶음에
`.py`, `.pyc`, `.pyo`가 포함되지 않는 것을 확인했다.

공개 저장소의 영문 `README.md`와 한국어 번역 `README.ko.md`에서 DMG 이름,
실행 스크립트, 라이선스, 체크섬 및 배포 안내를 검증한다. 동일한 README를
복사하던 `github/` 폴더는 공개 저장소를 간결하게 유지하기 위해 제거했다.

무료 오픈소스 포트폴리오 배포를 위해 Microsoft MarkItDown MIT 전문, Qt/PySide LGPLv3와
GPLv3 전문, 44개 배포 구성요소의 버전별 라이선스 묶음, LGPL 대응 소스 및 재링크 안내를
추가했다. 사용하지 않는 GPL 전용 Qt Virtual Keyboard와 연결 모듈을 두 앱에서 제거하고
끊어진 심볼릭 링크가 없는지 확인했다. 새 한국어·영문 DMG의 체크섬, 마운트, 라이선스 파일,
대응 소스 3개, 코드서명과 앱 실행을 모두 검증했다.
최종 DMG는 언어별 약 86MB이며, 대응 소스 아카이브는 합계 약 67MB다.

Microsoft 상표 지침에 맞춰 앱 브랜드를 `Batch Markdown Converter`로 변경하고 앱 이름,
DMG, 번들 ID, 설정 저장소, 언어 환경변수, 문서와 배포 명령을 함께 갱신했다. 시스템 이모지로
표시하던 뼈다귀도 자체 `QPainter` 벡터로 교체했다. 변경 후 의존성 무결성 검사와 전체 80개
테스트를 다시 실행해 모두 통과했다.

최종 출처 감사에서 Python 3.12 런타임과 PyInstaller bootloader를 42개 Python 의존성 목록과
별도로 식별하도록 보강했다. 각 DMG에 정확한 Python PSF 라이선스와 PyInstaller `COPYING.txt`,
버전, 원본 프로젝트 링크, Bootloader Exception 및 run-time hook의 Apache-2.0 고지를 포함한다.
